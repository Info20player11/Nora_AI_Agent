# Contributing to Nora

## Supported Contribution Types

We welcome contributions that improve Nora’s intelligence, security, and user experience. Please check which areas are currently open for community development.

| Category | Supported          |
| -------- | ------------------ |
| Core Logic (AI) | :white_check_mark: |
| UI/UX Design    | :white_check_mark: |
| Financial APIs  | :x: (Internal Only)|
| Bug Fixes       | :white_check_mark: |

## Submitting a Contribution

To maintain the financial integrity of Nora, we follow a strict contribution pipeline. Please follow these steps to ensure your code is accepted:

1. **Open an Issue:** Before starting any work, open an issue to discuss the proposed change. This prevents overlapping work and ensures the feature aligns with Nora's roadmap.
2. **Fork and Branch:** Fork the repository and create a feature branch (e.g., `feat/predictive-savings`).
3. **Financial Accuracy:** Any contribution involving mathematical calculations or financial logic must include unit tests with a minimum of 95% coverage. 
4. **Submit a Pull Request:** Once your changes are ready, submit a PR against the `main` branch. 
5. **Code Review:** Expect a technical review within **3-5 business days**. We may ask for adjustments regarding performance or data privacy.



**Note on Security:** If your contribution involves sensitive data handling or authentication, it will undergo an additional security audit before being merged into a stable release.
